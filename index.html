Overview

DocFlowEditor mounts onto any <div> and gives you a complete toolbar, content area, and status bar — no build step, no npm, no bundler. It ships with 25+ independent tools across four categories, supports custom tools, and has zero runtime dependencies.

Modular

Every tool is an independent module. Include only what your users need.

Split mounting

Mount toolbar and content in separate containers — handy for sticky toolbars.

Themeable

Light, dark, or OS-synced — switched instantly at runtime via setTheme().

Document-oriented

Page margin presets, borders, multi-column layout — beyond a simple textarea.

Events & API

React to changes via callbacks or DOM events; control content via the public API.

Extensible

Register custom tools with registerTool() using a simple interface.

Installation

Served from GitHub via jsDelivr — no build step, no npm. editor.js is an ES module that resolves its own tool imports relative to its CDN URL automatically.

Replace placeholders: swap USERNAME/REPO@v1.0.0 with your actual GitHub path and release tag once published. Pin a tag — never a branch — so the CDN link stays stable.
html 
<!-- 1. Stylesheet -->
<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/gh/USERNAME/REPO@v1.0.0/editor.css">

<!-- 2. ES module -->
<script type="module">
  import { DocFlowEditor } from
    'https://cdn.jsdelivr.net/gh/USERNAME/REPO@v1.0.0/editor.js';
</script>

Requires native ES module support — all current evergreen browsers. IE is not supported.

Quick Start

The minimal setup: point DocFlowEditor at an empty container element. It replaces it with a toolbar, content area, and status bar in one call.

html + js 
<!-- The container -->
<div id="my-editor"></div>

<script type="module">
  import { DocFlowEditor } from './editor.js';

  const editor = new DocFlowEditor('#my-editor');
</script>

The first argument accepts a CSS selector string or a DOM element reference directly.

Split Mode

Mount the toolbar and content area in separate containers — useful for sticky toolbars, full-page layouts, or when you need the editor areas in different DOM positions.

javascript 
import { DocFlowEditor } from './editor.js';

// Pass targets as a two-element array: [toolbar, content]
const editor = new DocFlowEditor(['#toolbar-slot', '#content-slot']);
javascript 
import { DocFlowEditor } from './editor.js';

// Or use an object form for clarity
const editor = new DocFlowEditor({
  toolbar: '#toolbar-slot',
  content: '#content-slot',
});
In split mode, destroy() restores each container's original HTML independently.

Configuration Reference

Pass a config object as the second constructor argument to customize the editor. All options have sensible defaults so you can start with an empty object.

javascript 
new DocFlowEditor('#my-editor', {
  tools: null,                  // null = all built-ins; or an array — see Toolbar Customization
  placeholder: 'Start writing…',
  minHeight: 297,               // mm — maps to --rte-min-height CSS variable (A4 by default)
  onChange: (html) => {},       // fires on every edit
  initialContent: '',           // seed HTML for the editor on mount
  theme: 'light',               // 'light' | 'dark' | 'auto' (follows OS)
  showStatusBar: true,          // false hides the word-count footer
});
Option Type Default Description
tools string[] | null null Subset & order of toolbar tools. null loads all built-ins.
placeholder string 'Start writing…' Shown when the content area is empty.
minHeight number 297 In millimetres. Defaults to A4 page height. Sets --rte-min-height.
onChange fn(html) null Callback fired on every keystroke or edit. Receives the current HTML string.
initialContent string '' HTML to pre-populate the editor. Falls back to the container's existing markup.
theme 'light' | 'dark' | 'auto' 'light' 'auto' syncs with the OS prefers-color-scheme media query.
showStatusBar boolean true Toggles the word and character count footer.

Toolbar Customization

Pass tools as an explicit array. Order in the array is the order tools appear in the toolbar. Unrecognized names are silently skipped. To trim or reorder, just change the array — there's no separate exclude list.

javascript 
const editor = new DocFlowEditor('#my-editor', {
  tools: [
    'undoRedo',
    'bold', 'italic', 'underline',
    'heading', 'alignment',
    'orderedList', 'unorderedList',
    'link', 'image', 'table',
    'clearFormatting',
  ],
});

Registering a Custom Tool

A tool module is a plain JavaScript object with three keys: name, createButton(), and execute(). Register it before constructing the editor so it's included on first render.

javascript 
const insertDateTool = {
  name: 'insertDate',             // unique name — used in the tools array

  createButton(editor) {          // required — returns the toolbar HTMLElement
    const btn = document.createElement('button');
    btn.type = 'button';
    btn.textContent = '📅';
    btn.title = "Insert today's date";
    btn.addEventListener('click', () => insertDateTool.execute(editor));
    return btn;
  },

  execute(editor) {               // required — runs the action
    editor.contentArea.focus();
    document.execCommand('insertText', false, new Date().toLocaleDateString());
    editor.emitChange();
  },

  updateState(btnEl) {            // optional — called on selection change
    // Toggle an active class based on selection state, if relevant
  },
};

// Register, then construct
const editor = new DocFlowEditor('#my-editor', {
  tools: ['undoRedo', 'bold', 'insertDate'],
});
editor.registerTool(insertDateTool);
If you register a tool after the editor has already mounted, call editor._buildTools(); editor._buildUI(); to rebuild the toolbar.

Public API

All methods are available on the editor instance returned by new DocFlowEditor().

getHTML() → string Returns the current content as an HTML string.
setHTML(html) → void Replaces all content; re-wires any tables in the new HTML.
getText() → string Returns plain-text content (no HTML tags).
clear() → void Empties the content area completely.
focus() → void Programmatically focuses the content area.
destroy() → void Tears down the editor and restores the original DOM node(s).
setTheme(theme) → void Switch between 'light' and 'dark' at runtime, without remounting.
registerTool(module) → void Add a custom tool to the registry.
syncToolbarState() → void Manually re-sync toolbar active/inactive states with the current selection.
javascript 
// Quick reference
editor.setHTML('<p>Hello <b>world</b></p>');
console.log(editor.getHTML());
console.log(editor.getText());
editor.setTheme('dark');
editor.clear();
editor.focus();
editor.destroy();

Listening for Changes

Two equivalent ways to react to edits — use whichever fits your architecture:

javascript 
// 1. Config callback — fires on every keystroke/edit
const editor = new DocFlowEditor('#my-editor', {
  onChange: (html) => console.log('changed:', html),
});

// 2. DOM event — bubbles, so you can listen on a parent element too
editor.root.addEventListener('rte:change', (e) => {
  console.log('changed:', e.detail.html);
});

Theming

The editor container gets class="rte-editor", data-theme="light|dark", and a --rte-min-height CSS variable. Override the CSS variables in editor.css to style against these hooks.

Light mode
Default. Clean white surfaces, indigo primary.
Dark mode
Deep neutral backgrounds, lifted primary tones.
css — customise in editor.css 
/* Target dark theme specifically */
.rte-editor[data-theme="dark"] {
  --paper: #1e1e1e;
  --ink:   #e8e8e8;
}

/* Respect reduced-motion for animated toolbar entries */
@media (prefers-reduced-motion: reduce) {
  .rte-toolbar * { animation: none !important; }
}
javascript — runtime toggling 
// Follow OS preference on init
new DocFlowEditor('#my-editor', { theme: 'auto' });

// Toggle at runtime — no remount needed
themeToggleBtn.addEventListener('click', () => {
  const next = editor.config.theme === 'dark' ? 'light' : 'dark';
  editor.setTheme(next);
  // Also update your page theme if needed:
  document.documentElement.setAttribute('data-bs-theme', next);
});
Tip: always call editor.setTheme() alongside your page-level theme toggle. This keeps the editor in sync without remounting it.

Split-target mounting, curated toolbar, registered custom tool, seeded content, runtime theme toggling, change events, and the full public API — all in one example.

// 1. Define a custom tool
const insertDateTool = {
  name: 'insertDate',

    const btn = document.createElement('button');
    btn.type = 'button';

    btn.title = "Insert today's date";
    btn.addEventListener('click', () => insertDateTool.execute(editor));
    return btn;
  },
  execute(editor) {
    editor.contentArea.focus();
    document.execCommand('insertText', false, new Date().toLocaleDateString());
    editor.emitChange();
  },
};

// 2. Construct in split mode with a curated toolbar
const editor = new DocFlowEditor(
  { toolbar: '#toolbar-slot', content: '#content-slot' },
  {
    tools: [
      'undoRedo',
      'bold', 'italic', 'underline', 'strikethrough',
      'heading', 'alignment', 'orderedList', 'unorderedList',
      'link', 'image', 'table', 'insertDate',
      'clearFormatting', 'findReplace',

    minHeight: 297,

    onChange: (html) => console.log('autosave payload:', html),
  }
);

editor.registerTool(insertDateTool);

// 3. Events and public API
editor.root.addEventListener('rte:change', (e) => {
  // e.detail.html === editor.getHTML()

// ✅ Correct: sync BOTH the page theme AND editor theme together
